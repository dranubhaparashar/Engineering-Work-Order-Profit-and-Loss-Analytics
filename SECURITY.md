# Security Policy

## Supported versions

The latest version on the default branch is supported for security updates.

## Reporting a vulnerability

Do not disclose a vulnerability or secret in a public issue. Report it privately to the repository owner through an appropriate private channel on GitHub.

Include:

- Affected file or component
- Reproduction steps using synthetic data
- Potential impact
- Suggested mitigation, when available

## Data exposure

This repository must remain synthetic-data-only. If real credentials or confidential records are discovered:

1. Remove public access where possible.
2. Revoke and rotate exposed credentials immediately.
3. Remove the material from the full Git history.
4. Review access logs and notify the appropriate data/security owner.
5. Document the incident through the organization’s approved process.
